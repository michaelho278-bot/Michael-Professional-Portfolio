📘 CMS README v5.0
## 🚀 專案簡介
本 CMS 系統採用 PHP 8 + OOP + MVC 架構，前後端分離設計，全面使用 JWT 驗證 控制登入與角色權限。
前端以 HTML/JavaScript + DataTables 呈現，後端以 API + Models + Helpers + Middleware 提供資料存取與安全驗證，確保系統可維護性、擴展性與安全性。

## 🎯 架構重點
三層分離：API → Model → Database Helper，清晰分工。
前後端分離：前端 JS (DataTable + CRUD) 與後端 API 對應。
安全性：所有 API 經過 ApiMiddleware 驗證 JWT。
擴展性：新增模組只需加 Model + API + JS + HTML。
可維護性：統一錯誤回傳格式，集中管理 DB 與 JWT。

## 🔑 功能特色 (v5.0 更新)
登入驗證：使用 LoginModel 驗證帳密，成功後生成 JWT。
JWT 驗證：所有 API 需帶 Authorization: Bearer <token>。
角色控制 (RBAC)：JWT payload 包含 role，前端根據角色顯示不同功能。
用戶管理：CRUD 操作（新增、更新、刪除、查詢）。
產品管理：CRUD 操作，支援 DataTables 顯示。
檔案上傳：新增 UploadHelper，支援產品圖片上傳。
系統紀錄：新增 LogModel + logs.html，紀錄 API 操作與登入事件。
安全性：密碼使用 password_hash() + password_verify()。

## 🔄 資料流流程
登入流程
前端 login.html  → 提交帳密 → api/login.php
後端驗證 → 生成 JWT → 回傳 {token, user}
前端存入 localStorage.token
受保護資源
前端呼叫 API → 帶 Authorization: Bearer <token>
後端 JwtHelper::verifyToken() → 驗證成功 → 呼叫 Model
Model 使用 Database 執行 SQL → 回傳資料
Controller echo json_encode(data) → 前端渲染
