# Token-based API (PHP + PDO + JWT)

## 📖 專案簡介
一個基於 PHP + MySQL 的 CMS 後端 API，採用 JWT 驗證，並逐步 OOP 化。
目前支援 Users 管理同 Products 管理，並提供基本登入 / 登出流程。

---

## 🚀 安裝步驟

1. **Clone 專案**
   ```bash
   git clone https://github.com/your-repo/cms-api.git
   ```

2. **建立資料庫**
   - 建立資料庫 `hgkdb`
   - 匯入 `users` 和 `products` 資料表

3. **安裝依賴**
   ```bash
   composer require firebase/php-jwt
   ```

4. **設定 config.php**
   - 資料庫連線設定
   - `JWT_SECRET` (至少 32 字元隨機字串)
   - `JWT_EXPIRY` (例如 3600 秒)

---

## 📂 專案結構

```
/api
 ├── config.php       # 資料庫連線 + JWT secret
 ├── auth.php         # JWT 生成 / 驗證
 ├── login.php        # 登入，生成 JWT
 ├── logout.php       # 登出 (前端清除 token)
 ├── user.php         # Users API (Admin CRUD / User GET 自己)
 ├── product.php      # Products API (Admin CRUD / User GET 全部)
 ├── models/
 │  ├── User.php      # Users Model (OOP)
 │  └── Product.php   # Products Model (OOP)

### 登入

```http
POST /api/login.php
Content-Type: application/json

{
  "uUsername": "admin",
  "uPassword": "123456"
}
```

**Response:**
```json
{
  "status": "success",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}
```

### Users API

| 方法 | 端點 | 說明 | 權限 |
|------|------|------|------|
| GET | `/api/user.php` | 取得用戶列表 | Admin: 全部用戶 / User: 自己 |
|  👥 權限控制

| 角色 | Users API | Products API |
|------|-----------|--------------|
| Admin | CRUD 全部 | CRUD 全部 |
| User | GET 自己 | GET 全部 |

---

## 🔒 登出流程

1. **前端清除 Token**
   - 刪除 localStorage / sessionStorage / cookie 中的 token

2. **可選：呼叫登出 API**
   ```http
   POST /api/logout.php
   Authorization: Bearer [token]
   ```
