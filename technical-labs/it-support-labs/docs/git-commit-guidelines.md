# Commit Message Guidelines

## Format
<type>: <short description>

### Common Types
- **add**: 新增檔案或功能  
- **update**: 更新現有內容  
- **fix**: 修正錯誤或排版  
- **refactor**: 重構結構或改善格式  
- **docs**: 文件相關更新 (README, template)  
- **chore**: 其他雜項 (例如路徑調整)

---

## Examples

- add: DHCP-DNS incident report  
- add: Printer issue report with screenshots  
- update: AD login incident evidence  
- fix: typo in DHCP-DNS report  
- docs: add incident-template.md skeleton  
- refactor: reorganize repo folder structure  
- chore: update .gitignore for screenshots
